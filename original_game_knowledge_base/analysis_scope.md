# Scope анализа оригинального кода

Какой код анализируется для KB, а какой — нет, и почему.

---

## Код НЕ анализируемый (переписывается с нуля)

| Система | Файлы/папки | Причина |
|---------|------------|---------|
| Рендеринг (DirectX 6) | `DDEngine/`, `poly*.cpp`, `aeng.cpp`, `Engine.cpp` | Заменяется на современный API |
| DirectDraw обёртка | `DDLibrary/` (частично) | Заменяется на SDL3 |
| Glide renderer | `Glide Engine/` | Мёртвый код, `VERSION_GLIDE` не определён |
| Software renderer | `sw.cpp` | Не нужен с аппаратным ускорением |

**Исключение:** В файлах рендеринга могут быть мутации game state (пример: `drawxtra.cpp` — `DRAWXTRA_MIB_destruct()` мутирует ammo). Такие места задокументированы в KB.

---

## Код НЕ анализируемый (платформы не переносим)

| Платформа | Файлы/папки | Причина |
|-----------|------------|---------|
| PlayStation 1 | `PSXENG/`, `#ifdef PSX` блоки | Не поддерживаем |
| Dreamcast | `#ifdef TARGET_DC` блоки | Не поддерживаем |
| Редактор уровней | `Editor/`, `GEdit/`, `LeEdit/`, `SeEdit/` | Не переносим |
| MuckyBasic VM | `MuckyBasic/` | Автономный инструмент, не интегрирован с игрой |

**Исключение PSX controls:** раскладка кнопок PSX контроллера задокументирована → [psx_controls.md](subsystems/psx_controls.md).

---

## Код НЕ анализируемый (не вошёл в финальную игру)

Подробности — в [cut_features.md](cut_features.md).

| Фича | Код | Источник решения |
|------|-----|-----------------|
| Мотоцикл (BIKE) | `bike.cpp`, `~15 файлов` | Mike Diskett: "never shipped" |
| Кошка-гарпун | `hook.cpp` | Mike Diskett: "never shipped" |
| Канализация | `sewer.cpp`, `ns.cpp` | Mike Diskett: "never shipped" |
| Процедурные интерьеры | `building.cpp` (частично) | Mike Diskett: "once upon a time" |
| Мультиплеер | `MAX_PLAYERS=2`, split-screen | В финале отключён |
| GS_REPLAY | Код записи/воспроизведения | В финале не используется |

---

## Код НЕ используемый в игре (мёртвый)

| Файл/система | Статус | Как обнаружено |
|-------------|--------|---------------|
| `MuckyBasic/` | Не интегрирован с движком | Нет механизма вызова из игрового кода |
| `briefing.cpp` | Мёртвый код (Dec 1998 прототип) | `#ifdef OBEY_SCRIPT` закомментирован |
| `cam.cpp` | Мёртвый (`#ifdef DOG_POO`) | Только `fc.cpp` активен |
| `morph.cpp` | Не используется | Код есть, не вызывается |
| `nd.cpp` | Пустой файл | — |
| `lead.cpp` | Недоделан | Верёвка/поводок не завершена |
| `Command.cpp` | Legacy NPC commands | Только `COM_PATROL_WAYPOINT` используется |

## Что АНАЛИЗИРУЕТСЯ

Всё остальное — активный игровой код для PC Release build. Основные подсистемы перечислены в [SESSION_START.md](SESSION_START.md) (карта подсистем).

**Конфигурация PC Release в пре-релизной кодовой базе** (из `Fallen.vcxproj`, Release конфигурация):
```
NDEBUG;_RELEASE;WIN32;_WINDOWS;VERSION_D3D;TEX_EMBED;FINAL
```
⚠️ Это конфигурация из пре-релизного snapshot, не из финального билда. Реальный релиз мог иметь другие/дополнительные флаги. `.vcxproj` мог быть обновлён при миграции проекта на новую VS (оригинал — VS6 `.dsp`).

Код под `#if 0`, `/* */`, `#ifdef EDITOR` и подобными — документируется как мёртвый, но не анализируется детально (если нет подозрений на активную логику внутри).
