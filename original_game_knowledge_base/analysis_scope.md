# Scope анализа оригинального кода

Какой код анализируется для KB, а какой — нет, и почему.

---

## Папки которые полностью пропускаются

См. [overview.md](overview.md) — структура папок с пояснением причин для каждой IGNORED папки.

**Исключение из рендеринга:** в файлах DDEngine могут быть мутации game state (пример: `drawxtra.cpp` — `DRAWXTRA_MIB_destruct()` мутирует ammo). Такие места задокументированы в KB.
**Исключение из PSX:** раскладка кнопок PSX контроллера задокументирована → [psx_controls.md](subsystems/psx_controls.md).

## Что АНАЛИЗИРУЕТСЯ

Всё остальное — активный игровой код для PC Release build. Основные подсистемы перечислены в [KB_INDEX.md](KB_INDEX.md).

**Эталон финальной игры: PC (Steam) И PS1.**
Фичи присутствующие в любой из версий — анализируются и переносятся.
При конфликте реализаций — берётся PC версия.

**PC-only фичи (нет на PS1) — анализируются и переносятся:**
- Лужи (`puddle.cpp`, `#ifndef TARGET_DC`)
- Bump mapping / Crinkle на ящиках — отключён в пре-релизе, работает в финальной PC
- Гильзы от выстрелов (`DIRT_create_brass`, prim 253)
- Rocket exhaust particles у платформ-PLAT

**PC-only фичи но отключены в финале — не переносятся:**
- Воздушные шары (`Balloon.cpp`) — отключены
- Симуляция ткани (`cloth.cpp`) — отключена
- Голубь (`PIGEON` в PYRO) — `prob_pigeon=0`, никогда не создаётся

**Конфигурация PC Release в пре-релизной кодовой базе** (из `Fallen.vcxproj`, Release конфигурация):
```
NDEBUG;_RELEASE;WIN32;_WINDOWS;VERSION_D3D;TEX_EMBED;FINAL
```
⚠️ Это конфигурация из пре-релизного snapshot, не из финального билда. Реальный релиз мог иметь другие/дополнительные флаги. `.vcxproj` мог быть обновлён при миграции проекта на новую VS (оригинал — VS6 `.dsp`).

Код под `#if 0`, `/* */`, `#ifdef EDITOR` и подобными — документируется как мёртвый, но не анализируется детально (если нет подозрений на активную логику внутри).
