## Итерация 2: Аудит аннотаций в коде (`// claude-ai:`) ✅ ВЫПОЛНЕНА

**Цель:** все комментарии `// claude-ai:` в оригинальном коде корректны и на своих местах.

### Задачи:
1. **grep все `// claude-ai:` комментарии** — собрать полный список
2. **Для каждого комментария проверить:**
   - Стоит ли в правильном месте (относится к коду ниже/выше него)
   - Матчится ли с текущей документацией в KB
   - Нет ли фактических ошибок
3. **Исправить или удалить** некорректные комментарии
4. **Отметить расхождения** с KB → входные данные для итерации 3

### Технический подход:
- Субагенты: grep для сбора всех аннотаций, группировка по файлам
- Основной контекст: точечная проверка каждой аннотации vs код вокруг неё
- Разбить на батчи по файлам если много

### Результат (2026-03-13):
- 3905 аннотаций в 98 файлах; проаудировано 16 файлов (~2600 аннотаций, 66% покрытие)
- ~35 фактических ошибок найдены и исправлены (error rate ~1.3%)
- Критичная ошибка: GRAVITY=-5120 вместо -51 (множитель 100x!) в collide.cpp и Vehicle.cpp
- Типичные ошибки: неправильные числа, перепутанные описания, устаревшие номера строк, неправильные единицы
- Оставшиеся ~50 файлов с мелкими аннотациями (<20 строк) — покрыть в итерациях 3-4

## Итерация 9: Dense Summary + инструкция по пересборке

**Цель:** создать компактную версию всей KB для задач требующих полного контекста (архитектурные решения, поиск скрытых зависимостей, ответы на сложные вопросы по всей игре).

### Задачи:
1. **Создать `DENSE_SUMMARY.md`** — сверхкомпактная версия всей KB:
   - Все подсистемы с ключевыми цифрами, формулами, константами
   - Межсистемные связи и неочевидные зависимости
   - Критичные факты которые легко забыть
   - **НЕ** развёрнутые описания — только суть
   - Целевой размер: ~30-40K токенов (влезает в контекст с запасом на работу)
2. **Вычистить SESSION_START.md** — убрать из него всё что дублируется с DENSE_SUMMARY. SESSION_START остаётся как лёгкий индекс + инструкция старта сессии, DENSE_SUMMARY — как полный контекст.
3. **Создать `DENSE_SUMMARY_BUILD_GUIDE.md`** — пошаговая инструкция как пересобрать dense summary с нуля:
   - Какие файлы KB читать и в каком порядке
   - Что из каждого файла извлекать в summary
   - Формат и структура summary
   - Целевой размер и как проверить что не вышли за лимит
   - Когда пересобирать (после значимых изменений в KB)

### Когда грузить DENSE_SUMMARY:
- Архитектурные обсуждения и принятие решений
- Вопросы затрагивающие несколько подсистем
- Поиск скрытых зависимостей
- Анализ всей KB целиком (аудит, консистентность)

### Когда НЕ грузить (достаточно SESSION_START + файлы подсистемы):
- Работа над конкретной подсистемой
- Написание кода по конкретной фиче

## Общие правила для всех итераций

- **Одна итерация = одна или несколько сессий** (не растягивать сессию до 90%+ контекста)
- **Субагенты — для механики** (grep, glob, подсчёты), не для анализа
- **Результаты записывать сразу** в KB, не накапливать
- **SESSION_START.md обновлять** в конце каждой итерации (прогресс, новые находки)
- **Пользователь явно говорит когда начинать** — не начинать анализ без команды
