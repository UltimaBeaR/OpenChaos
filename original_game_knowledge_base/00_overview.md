# Обзор архитектуры оригинальной игры

## Общая информация

- **Игра:** Urban Chaos (рабочее название: "Fallen")
- **Студия:** MuckyFoot Productions
- **Год:** 1999
- **Платформы:** PC (DirectX), PSX, Dreamcast
- **Язык:** C (несмотря на расширение .cpp)
- **Компилятор оригинала:** Visual Studio 6, DirectX 6/7

## Структура исходников

```
original_game/
├── fallen/              — основная игра
│   ├── Source/          — 184 .cpp файла, игровая логика
│   ├── Headers/         — 144 .h файла
│   ├── DDEngine/        — движок рендеринга и анимации
│   ├── DDLibrary/       — низкоуровневая работа с DirectX, DirectInput
│   ├── Editor/          — редактор карт и анимаций (НЕ НУЖЕН в новой версии)
│   ├── GEdit/           — часть редактора карт
│   ├── Ledit/           — редактор анимаций
│   ├── PSXENG/          — PSX-специфичный код (НЕ НУЖЕН)
│   └── Debug/           — MIX: артефакты сборки (.obj, .exe, .dll, .pdb) + ресурсы игры в подпапках
├── MFLib1/              — кросс-платформенная графическая библиотека студии
├── MFStdLib/            — стандартные утилиты (файлы, память, математика)
├── MuckyBasic/          — собственный скриптовый язык (VM + lexer + parser)
└── thrust/              — побочный проект, не относится к Urban Chaos
```

## Ключевые архитектурные паттерны

### 1. Thing — базовый "класс" всех объектов

Все игровые объекты (персонажи, машины, здания, предметы, снаряды и т.д.) —
это `Thing`. Полиморфизм реализован через union + function pointers, не через C++ наследование.

```c
// Упрощённо
typedef struct Thing {
    SLONG Type;           // PLAYER, PERSON, VEHICLE, BUILDING, SPECIAL, ...
    void (*StateFn)(Thing*);  // функция-обработчик текущего состояния
    Thing *Parent, *Child, *LinkParent, *LinkChild;  // linked list иерархия
    SLONG Flags;          // 50+ битовых флагов состояния
    union {               // данные специфичные для типа
        PersonStuff person;
        VehicleStuff vehicle;
        // ...
    };
} Thing;
```

### 2. MapWho — пространственный хэш

Паттерн от Bullfrog (разработчики Dungeon Keeper, Theme Park и т.д.).
2D массив ячеек, к каждой из которых прикреплён linked list объектов в этой области.
Используется для быстрого spatial lookup — "какие объекты рядом с этой точкой".

### 3. Процедурная генерация геометрии

Здания, заборы, пожарные лестницы — не хранятся как готовые меши,
а генерируются процедурно из wall-данных во время загрузки/рендеринга.

### 4. Heightmap карта

Карта мира — heightmap (2D массив высот) с возможностью вертикального смещения
для моделирования бордюров. См. `pap.h`.

### 5. Компиляционные платформы

Один и тот же код компилируется под 3 платформы через препроцессор:
- `#ifdef PSX` — PlayStation 1
- `#ifdef TARGET_DC` — Dreamcast
- `#ifdef VERSION_D3D` — PC DirectX

Для новой версии нужен только PC-путь. PSX/DC код полностью выпиливается.

## Что НЕ нужно в новой версии

- Поддержка PSX (`#ifdef PSX` блоки)
- Поддержка Dreamcast (`#ifdef TARGET_DC` блоки)
- Редактор карт (Editor/, GEdit/, Ledit/)
- Редактор анимаций (Ledit/)
- Glide renderer (3dfx Voodoo карты) — в оригинале есть отдельная Glide Engine реализация
- Software renderer (sw.cpp — был fallback для машин без 3D ускорителя)
- Экспериментальные фичи, не вошедшие в релиз (мотоциклы, кошка-крюк — проверить)
- MuckyBasic (заменить на современный скриптовый язык — TBD)

## Ресурсы игры

Находятся в подпапках `original_game/fallen/Debug/` (Steam-версия, рабочие).
Сама папка Debug — это микс: артефакты сборки (.obj, .exe, .dll, .pdb, .log) + ресурсы в подпапках.

**Подпапки с ресурсами:**
- `Meshes/` — 3D меши
- `Textures/` — текстуры
- `clumps/` — clump-файлы (3D модели персонажей/объектов)
- `data/` — игровые данные
- `levels/` — файлы уровней
- `talk2/` — диалоги
- `text/` — текстовые строки
- `bink/` — Bink-видео (катсцены)
- `stars/` — текстуры звёздного неба
- `server/` — (уточнить что это)
- `config.ini` — конфигурационный файл игры (из Steam-версии)

**Артефакты сборки (не нужны):** все .obj, .exe, .dll, .pdb, .ilk, .log файлы в корне Debug/
