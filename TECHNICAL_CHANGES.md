# Technical Changes — OpenChaos vs Original

Major technical and architectural changes made to the codebase, relative to the **original pre-release MuckyFoot sources** the project started from — not relative to the retail release.

---

## Build & toolchain

- Compiler switched from MSVC to Clang (standalone `clang++`).
- Build system replaced Visual Studio projects with CMake + Ninja; dependencies managed via vcpkg; C++20.
- Cross-platform build driven by a Makefile with per-platform CMake toolchain files.

## Architecture & platform

- Full port from 32-bit to 64-bit (x64).
- Builds as a single static one-file executable (no DLLs needed to run).
- Cross-platform: Windows (x64), macOS (Apple Silicon), and Linux / Steam Deck.
- The application now runs entirely on SDL3 — windowing and input go through SDL3.

## Rendering

- Renderer replaced: Direct3D 6 → OpenGL 4.1 core.
- Fixed-function pipeline reimplemented as GLSL shaders.
- Full skeletal animation: characters (bodies, shadows, and puddle reflections) are now GPU-skinned — real skeletal animation rather than the original's approximation of it.
- Resolution unlocked — no longer hardcoded to 640×480 4:3: windowed and fullscreen modes, with support for different screen aspect ratios. The UI is adaptive/responsive in places, adjusting to the resolution and aspect ratio.
- Scene rendered into an offscreen FBO with a composition pass — the foundation for further post-processing (the CRT/scanline shader is one example built on top of it).
- Puddle reflections / wibble reworked.
- FPS unlocked: physics (fixed at 20 Hz) is fully decoupled from rendering (uncapped frame rate), backed by a substantial render-interpolation system that smoothly interpolates entity positions and skeletal poses between physics ticks and correctly handles teleports and other discontinuities.
- Significant rendering-performance optimization work was done, getting the game up to stable high frame rates — 90+ FPS on the Steam Deck.

## Audio

- Audio backend replaced: Miles Sound System (MSS32) → OpenAL Soft (3D positional audio preserved).

## Input & controllers

- Full gamepad support added (Xbox/generic via SDL3, plus DualSense): hotplug, automatic input-device detection, and rumble.
- DualSense: our own `libDualsense` library, which we implemented ourselves (on SDL3 + hidapi) from pieces of other libraries and the protocol specs, instead of relying on a vendored third-party lib — driving the LED lightbar and adaptive triggers.

## Video

- Cutscene video playback reimplemented on FFmpeg instead of the proprietary Bink SDK.

## Config & user data

- New JSON config system (`OpenChaos.config.json`) replacing `config.ini` (the localization language is still read from `config.ini` for retail compatibility). All writable files (saves, config, logs, screenshots) are redirected to an OS per-user data folder (via `SDL_GetPrefPath`), with read-fallback to the game folder — which is now treated as read-only.

## UI & fonts

- Built-in English fonts for English-only / non-resource text, generated by us in code from a public-domain base font (not extracted from the game's own font assets — so they're license-clean), baked into our own atlas at startup.
- Self-contained "files not found" / "language files not found" startup screens, rendered from an embedded font with no external resources.
- In-game Help menu (in the pause menu) showing the control layout and descriptions of some game mechanics, with added button glyphs (icons) for keyboard and gamepad buttons.

## Source restructuring & cleanup

- Full restructuring of the source tree into a modern domain-based hierarchy (`engine/`, `game/`, …), with every entity traced back to the original (`uc_orig` comments + an entity map).
- All graphics-API (OpenGL) calls isolated behind a rendering abstraction and concentrated in one place.
- Several iterations of cleaning out old and unused code — all (or nearly all) unused code, plus all experimental and debug-only functionality behind define flags, has been removed: anything not used was deleted.
