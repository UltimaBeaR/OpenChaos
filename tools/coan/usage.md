# coan — документация и инструкция использования

**coan** (The C Preprocessor Chainsaw) — утилита для статического анализа и упрощения C/C++ кода
на основе конфигурации препроцессора.

Исполняемый файл: `tools/coan/coan.exe` (Windows x64)
Официальный сайт: https://coan2.sourceforge.net/

---

## Что делает

Принимает набор флагов `-D` (определить макрос) и `-U` (undefinить макрос), статически вычисляет
все `#if`/`#ifdef`/`#ifndef` условия в исходниках и удаляет мёртвые ветки вместе с директивами.

Корректно обрабатывает сложные условия:
- `#if !defined(TARGET_DC) && defined(_DEBUG)` → при `-UTARGET_DC` упрощается до `#ifdef _DEBUG`
- `#if defined(PSX) || defined(TARGET_DC)` → при `-UPSX -UTARGET_DC` весь блок удаляется
- Вложенные `#ifdef` — обрабатывает правильно

---

## Синтаксис команды

```
coan source [флаги] --no-transients --filter cpp,c,h,hpp --recurse <папка>
```

`source` с `--recurse` редактирует файлы на месте (in-place). Бэкап — через git (`git diff` после запуска).

---

## Флаги

| Флаг | Значение |
|------|----------|
| `-DSYMBOL` | Считать SYMBOL определённым |
| `-USYMBOL` | Считать SYMBOL неопределённым (удалить его ветки) |
| `--no-transients` | Игнорировать `#define`/`#undef` внутри самих файлов — смотреть только на флаги командной строки |
| `--filter cpp,c,h,hpp` | Обрабатывать только эти расширения |
| `--recurse` | Рекурсивно по папкам |

**`--no-transients` нужен всегда** в нашем проекте: мы задаём конфигурацию явно,
внутрифайловые define'ы не должны на неё влиять.

---

## Коды выхода

Exit code 19 при использовании `--no-transients` — норма. Это просто предупреждение
"Use at your own risk", не ошибка. Файлы обрабатываются корректно.

---

## После каждого запуска coan

Обязательно выполнить `git status` и проверить нет ли файлов вида `coan_out_NNNNNN`.
Они появляются когда coan не смог обработать файл до конца (например, orphan `#endif`) и не переименовал
временный выходной файл в оригинал. Такие файлы — мусор, удалять:

```
find new_game -name "coan_out_*" -delete
```

## Известные риски и нюансы

- **Include-файлы не парсятся** — define'ы из заголовков coan не видит. Не проблема: все
  нужные флаги передаются явно через `-D`/`-U`.
- **Макросы не разворачиваются** — `#if SOME_MACRO > 0` где `SOME_MACRO` определён через другой
  макрос coan может не вычислить. При обнаружении таких мест — фиксить вручную после прогона.
- **Лишние пустые строки** — после удаления блоков coan может оставить пустые строки. Некритично.

## Полезные команды диагностики

Посмотреть все директивы препроцессора в проекте (без изменений):
```
tools\coan.exe symbols --ifs --once-only --gag error --filter cpp,c,h,hpp --recurse new_game/fallen
```
