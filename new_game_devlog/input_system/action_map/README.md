# Action Map — упрощённая модель семантических констант ввода

Это **упрощённая** реализация идеи action-системы из
[`../full_plan_deferred.md`](../full_plan_deferred.md). Полноценный action-runtime
(маппинг, ремап-UI, контексты-стек, reverse-lookup glyph'ов) **не** делается —
сейчас цель скромнее:

> Все обращения к клавишам/кнопкам/осям в коде идут не через сырые scancode/индексы,
> а через семантические `ACT_*` константы. По одному поиску `ACT_FOOT_ATTACK_` сразу
> видно ВСЕ типы устройств, привязанные к этому действию. Замена биндинга = правка
> одной строки. Никакого рантайма поверх не нужно.

Параллельно убирается старая система переназначения клавиш (env + UI экран) и
gamepad→keyboard bridge — они ломают семантику compile-time констант.

## Связь с parent input_system

`input_frame` API (шапка [parent README](../README.md)) — это **уровень железа**:
`input_key_held(KKEY_SPACE)`, `input_btn_just_pressed(GBTN_SOUTH)` и т.д. Тип
параметра — сырой scancode/btn-index.

`action_map` — это **семантический уровень поверх**: вместо `KKEY_SPACE` в код
пишется `ACT_FOOT_JUMP_KKEY`, которая разворачивается в `KKEY_SPACE` через
constexpr. На рантайм оверхеда ноль.

## Релиз 1.0

В релизе 1.0 переназначение клавиш в UI **заблокировано** — все биндинги хардкод
через эти константы. Внутри игры будет статичная справка «какая клавиша что
делает» (вне скоупа этой работы). Полноценный runtime-ремап вернётся после 1.0
если потребуется — тогда `ACT_*` константы заменятся на runtime-resolver, точки
вызова в коде не меняются.
