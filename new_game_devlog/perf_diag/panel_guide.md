# perf_diag — инструкция по панели (что меряет, как пользоваться)

Памятка для чтения панели во время отладки перфа. Базовая версия панели
готова. Архитектура/история — [`design.md`](design.md).

## Как включить

Флаги в `new_game/src/debug_config.h` (dev-only, в коммит/релиз — `false`):

- `OC_DEBUG_PERF` — сама панель + хоткеи.
- `OC_DEBUG_PERF_LOG` — запись CSV в файл (можно включать отдельно, ловит
  фазу загрузки с самого старта процесса).

Размер шрифта панели — **не флаг**, а кодовая константа
`DBG_PANEL_TEXT_SCALE` в `new_game/src/engine/debug/debug_panel_text_scale.h`
(общая с dbglog). 1 = мелкий 5×9, 2 = крупный 2×.

Пересобрать. Панель — слева, поверх HUD. Работает в геймплее, в меню и в
катсценах одинаково (рисуется отдельным пиксельным слоем).

## Хоткеи (геймплей, без bangunsnotgames)

| Клавиша | Действие |
|---------|----------|
| `4` | Показать / скрыть панель |
| `5` | Переключить окно усреднения: 10 → 30 → 60 → 120 кадров (по кругу). Это же значение становится окном усреднения файл-лога. |
| `6` | Сбросить накопленную историю/средние (обнулить статистику) |

Шапка панели: `FPS NNN (N last frames)` — N это текущее окно усреднения
(сколько последних кадров усредняется; меняется клавишей `5`).

## Что меряет каждая строка

Имена иерархические (`группа.часть`). Группы разделены пустой строкой,
внизу каждой группы — синтетическая строка `группа .total` с толстой
яркой линией = доля этой группы в кадре.

> ⚠️ **Важно про CPU vs GPU в нашем рендере (OpenGL).** Замеры —
> wall-clock на **CPU-потоке**. GPU работает асинхронно: вызовы
> отрисовки только ставятся в очередь драйвера (стоит CPU), а реальная
> работа GPU выполняется позже. Поэтому стоимость GPU **в основном не
> видна** в `render.draw_screen`/`overlay` (там почти чистый CPU:
> построение геометрии + сабмит), а «всплывает» в `render.present` —
> именно там CPU **блокируется**, ожидая, пока GPU домолотит кадр и
> придёт vsync. Практический вывод: высокий `draw_screen` = упёрлись в
> **CPU** (построение/сабмит/куллинг); высокий `present` = упёрлись в
> **GPU** или ждём vsync. Проценты GPU ниже — грубая прикидка для нашего
> пайплайна, не точный профиль GPU.

| Строка | Что меряет | CPU / GPU |
|--------|------------|-----------|
| `FPS NNN` (шапка) | Кадров в секунду (из полного времени кадра `frame.total`). | Итог: весь CPU-поток за кадр, **включая** ожидание GPU/vsync в present и сон в idle. |
| `physics.total` | Весь блок физики за кадр (может прогоняться 0..N тиков — catch-up при просадке): `process_things`, частицы, OB/TRIP/DOOR, скрипт миссии (EWAY), DIRT, гранаты, MAP/POW/FC/VC + снятие интерп-снапшотов. | **100% CPU.** GPU не задействован. |
| `physics.ticks` | Счётчик: сколько тиков физики прогналось в этом кадре. 1 — норма. >1 — машина не успевает, физика догоняет (явный признак просадки). | Счётчик (не время) — N/A. |
| `render.draw_screen` | `draw_screen()` → `AENG_draw` — построение/сабмит всей 3D-сцены. | **≈85% CPU / ≈15% GPU** (CPU: куллинг, сборка геометрии, смена стейтов, выдача draw-вызовов; ~GPU-часть — синхронные стоны драйвера). Реальная отрисовка GPU оплачивается позже в present. |
| `render.overlay` | `OVERLAY_handle()` — HUD, консоль, меню, пост-композишн UI. | **≈95% CPU / ≈5% GPU** (CPU: сборка 2D + сабмит). |
| `render._perf_panel` | Стоимость отрисовки самой этой панели. Инфо-подметрика (вложена в `render.overlay`), в сумму группы НЕ входит (иначе двойной счёт). Если тут много — тормозит сам диагностический оверлей. | **≈100% CPU** (сборка текста/квадов + сабмит). |
| `render.present` | `screen_flip()` — своп буферов + ожидание GPU/vsync. | **≈10% CPU / ≈90% GPU+vsync-ожидание.** Здесь CPU блокируется, ждёт пока GPU доделает кадр и придёт развёртка. Высокое значение = упёрлись в GPU или vsync, а не в CPU. |
| `render.frontend` | Только в главном меню: `FRONTEND_loop()` (логика+отрисовка меню). | **≈90% CPU / ≈10% GPU** (как draw_screen, но для меню). |
| `idle.wait` | `lock_frame_rate()` — намеренный сон чтобы держать FPS-кап. | **Ни CPU, ни GPU** — это намеренный сон потока (ожидание), не работа. Большое значение = есть запас, кадр успевает раньше кэпа. |
| `io.level_init` | `game_init()` — разовая загрузка уровня. В установившемся геймплее ≈0; большой всплеск в момент загрузки. | **≈80% CPU/диск / ≈20% GPU-upload** (чтение/парсинг ассетов + заливка текстур/буферов на GPU). |
| `группа .total` | Синтетика: доля группы в кадре. Для `physics` — измеренный блок целиком; для `render` — сумма его листьев (draw_screen+overlay+present). | Наследует от состава группы (physics — CPU; render — в основном CPU-сабмит + GPU-ожидание в present). |
| `(other) .total` | Немеряный остаток = 100% − сумма всех групп. Это зазоры между инструментированными блоками (ввод, звук, вывод на геймпад, прочее). | **CPU** (немеряная CPU-работа между блоками). |

**Сумма всех `.total` (включая `(other)`) = ровно 100% кадра.** Для общей
картины достаточно смотреть только строки `.total` (у них толстая яркая
линия); внутрь группы лезть только если её тотал подозрительный.

## Цвета

- Колонка процента красится по нагрузке: бело-серый → красный,
  нелинейно (от ~10% уже заметно тонировано, 100% — красный, 50 vs 80
  различимы). Так боковым зрением видно где «горит».
- Строки `.total` рисуются теми же цветами, что обычные. Единственное
  отличие тотала — **толстая яркая линия** под строкой.
- Под каждой строкой тонкая линия — чтобы не путать имя с далеко
  стоящим значением.

## Файл-лог (`OC_DEBUG_PERF_LOG`)

- Пишется `perf_log.csv` рядом с рабочей директорией процесса.
- Строка раз в N кадров (N = окно усреднения, по умолчанию 60; следует
  за клавишей `5`), значения усреднены за это окно. Per-frame не пишем —
  не успевает и тяжело анализировать.
- Колонки = имена метрик; заголовок перевыпускается если появились новые
  метрики (ленивая регистрация).
- Ловит фазу загрузки (флаг активен со старта процесса, до появления
  панели/хоткеев).
- Перед анализом конкретной проблемы режим усреднения подбирается под
  неё (для спайков — маленькое окно, для трендов — большое).

## Тюнинг (одним числом, в коде панели)

- `DBG_PANEL_TEXT_SCALE` (engine/debug/debug_panel_text_scale.h) — размер
  шрифта (общий с dbglog).
- `PERF_COL_GAP_PX` (perf_diag.cpp) — зазор между именем и значением.
- `PERF_TOP_PX` / `PERF_LINE_PX` — отступ сверху / шаг строки.

Ширина панели и колонка значения — НЕ хардкод: меряются по факту от
самого длинного имени каждый кадр (панель сама обтягивает контент).
