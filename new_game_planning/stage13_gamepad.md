## Быстрый выбор оружия: D-pad

**Идея:** D-pad напрямую (без модификатора) — мгновенный выбор оружия по слоту, минуя перелистывание. R3 **не трогаем** — оставляем текущее поведение (циклический перебор всего инвентаря) как есть.

**Раскладка:**
| D-pad | Оружие |
|-------|--------|
| ↑ | Пистолет |
| ← | Автомат (AK47) |
| → | Дробовик |
| ↓ | Ближний бой / кулаки (см. ниже) |

**Логика ↑ / ← / →:**
- Если нужное оружие есть в инвентаре и не в руках → мгновенное переключение, без анимации листания.
- Если оружие уже в руках → re-equip не делаем (не переигрываем draw-анимацию), но popup всё равно показывается как подтверждение нажатия.
- Если оружия нет в инвентаре → переключения не происходит, но popup показывает **текущее** оружие (визуальный feedback что кнопка нажалась — без него игрок не понимает, среагировала ли игра).

**Логика ↓ (ближний бой):**
- Если в руках любое оружие **кроме** ближнего → переключение на **пустой кулак**.
- Если в руках уже кулак или ближнее оружие → цикл по melee-слотам в порядке: **кулак → бита → нож → кулак → …** (циклически, то что нет в инвентаре — пропускается; кулак есть всегда).
- Popup показывается на каждое нажатие (так же как для ↑/←/→).

**Popup HUD:** на любое D-pad нажатие открывается тот же popup, что и для R3 (живёт ~3 сек, потом фейдится). В нём подсвечивается слот, который выбран D-pad'ом (или текущий слот, если оружия нет в инвентаре).

**R3 (без изменений):** циклический перебор всего инвентаря, ровно как сейчас. Все оружия, не назначенные на D-pad (граната, C4 и т.п.), по-прежнему доступны через R3. Код пути R3 **не трогаем**.

**Замечание по названию:** в исходниках оружие называется `AK47` (`PRIM_OBJ_WEAPON_AK47`), хотя визуально это M16. В коде и доке оставляем «AK47» как в оригинале, чтобы не плодить расхождений.

**Реализация:** обрабатывается в `game_tick.cpp` рядом с R3-путём (тот же гейтинг — `can_darci_change_weapon` + не на паузе). При `dpad_just_pressed(dir)` вызывается select-by-type (для ↑/←/→) или melee-toggle (для ↓), используя `CONTROLS_set_inventory` для эквипа — то же что R3-путь. Гейтится по чит-комбо (Select+L1+L2) чтобы не дублировать смену оружия с D-pad-читами в `input_actions.cpp`. R3-путь не модифицируется.

## Stick drift в меню
- В главном меню нет deadzone для стиков — бесконечный скролл на контроллерах с drift
- В геймплее deadzone есть, в меню — нет
- **Это баг для 1.0**, не для 13 этапа — записан в [known_issues](known_issues_and_bugs.md) — "Бесконечный скролл меню (stick drift)"
