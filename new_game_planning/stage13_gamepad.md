## Быстрый выбор оружия: R3 + D-pad

**Идея:** нажатие правого стика (R3) в сочетании с направлением D-pad — прямой выбор конкретного оружия, минуя перелистывание.

Сейчас R3 = цикличный перебор оружий (следующее в списке). Предложение: добавить «быстрый слот» — R3 зажат → D-pad выбирает оружие напрямую.

**Раскладка 4 направлений:**
| D-pad | Оружие |
|-------|--------|
| ↑ | Пистолет (самое частое) |
| → | Дробовик |
| ↓ | Автомат |
| ← | Ближний бой (бита/нож — лучшее из имеющегося) **или** граната |

Вариант для ←: либо «лучшее ближнее оружие» (приоритет: бита > нож), либо граната если есть. Можно сделать тоглер между этими двумя режимами, но это усложняет. Проще — фиксировать ← за ближним боем, гранату оставить на отдельной кнопке.

**Логика выбора слота:**
- Если нужное оружие не поднято — ничего не делать (не переключаться на следующее попавшееся).
- Если поднято — мгновенное переключение без анимации листания.

**Приоритеты оружий популярности** (по геймплею): пистолет > дробовик ≈ автомат >> ближний. Раскладка ↑→↓ отражает это.

**Реализация:** обрабатывать в `input_actions.cpp` / `person.cpp` — при `R3 held + dpad_just_pressed(dir)` вызывать select-by-type вместо cycle-next. R3 без D-pad продолжает работать как сейчас (цикл).

## Stick drift в меню
- В главном меню нет deadzone для стиков — бесконечный скролл на контроллерах с drift
- В геймплее deadzone есть, в меню — нет
- **Это баг для 1.0**, не для 13 этапа — записан в [known_issues](known_issues_and_bugs.md) — "Бесконечный скролл меню (stick drift)"
