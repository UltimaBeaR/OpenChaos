### [2026-03-12] Стратегия: переписать vs рефакторить

**Решение:** Переписать с нуля, используя оригинал как документацию и эталон.

**Причина:** Рефакторинг ~500K строк C89-кода с монолитными файлами (Game.cpp = 53K строк)
крайне сложен — всё связано со всем. Переписывание с нуля даёт чистую архитектуру
и не медленнее при наличии оригинала как эталона.

**Альтернативы:** Инкрементальный рефакторинг оригинала — отклонено.

### [2026-03-12] Язык: C++23

**Решение:** C++23 с modules.

**Причина:** Физика и логика портируются механически без концептуального перевода.
Лучшая нативная игровая экосистема. C++23 modules убирают хедер-хаос.

**Альтернативы рассмотрены:**
- Rust — другая парадигма, болезненный порт C-логики
- Zig — нестабильный язык, маленькая экосистема
- C# — хуже для низкоуровневого контроля

### [2026-03-12] Физика: порт 1:1

**Решение:** Физика и коллизии портируются из оригинала 1:1.

**Причина:** Оригинальная физика определяет геймплей. Изменения дадут другие
ощущения от игры. Риск недопустим.

**Примечание:** Нужны флаги компилятора для 32-bit float consistency (`-mfpmath=sse -msse2`).

### [2026-03-14] Графика: OpenGL 4.1 vs Vulkan — на рассмотрении

**Решение:** Два кандидата, финальный выбор при старте Фазы 3.

**Причина:** macOS в списке платформ. Apple deprecated OpenGL в 2018, максимум на macOS — 4.1.
OpenGL 4.6 отпадает. Выбор между OpenGL 4.1 (проще, но deprecated) и Vulkan+MoltenVK (сложнее, future-proof).
Vulkan — официальный преемник OpenGL от Khronos. MoltenVK транслирует Vulkan в Metal на macOS,
используется в продакшене (Steam и др.).

**Аргументы за OpenGL 4.1:** меньше boilerplate, быстрее стартовать.
**Аргументы за Vulkan:** future-proof, единый API, не нужно переписывать если захочется улучшить графику.

### [2026-03-14] Звук: OpenAL Soft vs miniaudio — на рассмотрении

**Решение:** Два кандидата, финальный выбор при старте Фазы 3.
OpenAL Soft предпочтительнее по качеству 3D spatializer.

**Причина:** Оригинал использует MSS32 с позиционным 3D звуком (`MFX_play_xyz`).
OpenAL Soft — прямой современный аналог. miniaudio — проще интегрировать, но 3D базовый.

### [2026-03-14] Скриптинг: не нужен

**Решение:** Скриптовый движок убран из стека.

**Причина:** MuckyBasic в игре не используется. Вся миссионная логика реализована
через EWAY — data-driven система EventPoints (42 условия, 52 действия, бинарный .ucm формат).
Нужна только реализация EWAY движка на C++.

**Альтернативы рассмотрены:** QuickJS, Wren — не нужны.

### [2026-03-14] Архитектура: строгое разделение рендерера и game state

**Решение:** Рендерер только читает game state, никогда не модифицирует.

**Причина:** В оригинале рендерер мутирует state (drawxtra.cpp изменяет ammo,
создаёт PYRO/SPARK эффекты прямо в draw pass). Это архитектурный антипаттерн.
В новой версии — явное разделение: update phase записывает, render phase только читает.
