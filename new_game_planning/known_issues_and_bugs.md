## Отсутствующий функционал (вырезан при модернизации)

| Проблема | Описание | Статус |
|----------|----------|--------|
| Видеоролики не воспроизводятся | Bink SDK (RAD Game Tools) проприетарный, вырезан при модернизации (Kai, коммит `51959ff` — "make the solution actually build something", `bink.h` закомментирован т.к. SDK отсутствовал). `PlayQuickMovie()` в `display.cpp` — заглушка. В оригинале (`GDisplay.cpp:788`) использовался `BinkPlay()` для трёх интро-роликов (`FMV1a.bik`, `FMV1b.bik`, `FMV2.bik`) и кат-сцен (`FMV3_%d.bik`). Файлы .bik присутствуют в ресурсах финальной игры. **Решение:** FFmpeg (libavcodec) — имеет декодеры Bink Video и Bink Audio, .bik воспроизводится без конвертации. План реализации: (1) декодирование через FFmpeg API (открыть контейнер, читать пакеты, декодировать кадры), (2) вывод видео — кадр → GL текстура → полноэкранный квад (TL quad rendering уже есть), (3) вывод аудио — PCM → OpenAL (уже в проекте), (4) A/V синхронизация по pts, (5) управление — skip по кнопке, loop для фоновых роликов меню. Связанный стаб в OpenGL бэкенде: `ge_run_cutscene` (core.cpp) — заглушка, точка интеграции для плеера. | Реализовать |
