Не факт что актуальны для нашей кодовой базы — требуют проверки.

| Баг | Коммит (PieroZ repo) | Описание | Статус |
|-----|----------------------|----------|--------|
| OpenAL спам "Invalid source ID 0" | `c8b1d76` (PieroZ) | Спам сообщений в логах. Не видно без отладчика. | Проверить |
| Bat/Baalrog: выход за массив | `021c515` (PieroZ) | `BAT_STATE_TURRET_FOLLOW` (20) удалён, `BAT_STATE_NUMBER` уменьшен 21→20. Массив `BAT_state_name` был на 1 элемент короче → потенциальный краш на босс-уровнях. | Проверить при прохождении |

## Визуальные проблемы (замечены вживую)

| Проблема | Описание | Статус |
|----------|----------|--------|
| Здания появляются резко за пределами тумана | Дистанция появления зданий/объектов слишком маленькая — туман уже прошёл, а здание резко pop-in'ится. Возможно настраивается через ini/config. | Исследовать |
| Чёрные дырки на ближнем клиппинге | Объекты на краю фрустума (ближе к камере) отсекаются по полигонам — образуются чёрные пустоты. Как будто frustum/near-plane culling слишком агрессивный. | Исследовать |

## Отсутствующий функционал (вырезан при модернизации)

| Проблема | Описание | Статус |
|----------|----------|--------|
| Видеоролики не воспроизводятся | Bink SDK (RAD Game Tools) проприетарный, вырезан при модернизации (Kai, коммит `51959ff` — "make the solution actually build something", `bink.h` закомментирован т.к. SDK отсутствовал). `PlayQuickMovie()` в `display.cpp` — заглушка. В оригинале (`GDisplay.cpp:788`) использовался `BinkPlay()` для трёх интро-роликов (`FMV1a.bik`, `FMV1b.bik`, `FMV2.bik`) и кат-сцен (`FMV3_%d.bik`). Файлы .bik присутствуют в ресурсах финальной игры. Для восстановления: либо FFmpeg/libvlc для воспроизведения .bik, либо конвертация в открытый формат + свой плеер. | Реализовать |
