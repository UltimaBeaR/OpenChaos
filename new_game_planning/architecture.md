# Архитектура новой версии

## Статус: 🔲 Черновик — детализировать после анализа оригинала

## Принципы

1. **Движок отделён от игры** — в оригинале всё перемешано, это исправляем
2. **Модули вместо хедеров** — C++23 modules, никаких .h файлов
3. **Явные зависимости** — никаких глобальных состояний без крайней необходимости; зависимости передаются явно через параметры или конструктор
4. **Читаемость** — код понятен без комментариев, комментарии объясняют "почему"
5. **Кросс-платформенность** — никакого platform-specific кода в игровой логике

## Предполагаемая структура проекта

```
urban_chaos_new/
├── engine/              — движок (не знает об Urban Chaos)
│   ├── core/            — базовые типы, математика, утилиты
│   ├── renderer/        — графика (OpenGL/Vulkan абстракция)
│   ├── audio/           — звук
│   ├── input/           — ввод (клавиатура, мышь, геймпад)
│   └── resources/       — загрузка ресурсов
├── game/                — сама игра (использует движок)
│   ├── world/           — карта, здания, terrain
│   ├── entities/        — персонажи, транспорт, предметы
│   ├── ai/              — поведение NPC
│   ├── physics/         — физика и коллизии (порт из оригинала)
│   ├── missions/        — EWAY движок (EventPoints: 42 условия, 52 действия, .ucm формат)
│   ├── ui/              — интерфейс
│   └── main.cpp
├── CMakeLists.txt
└── vcpkg.json
```

## Ключевые архитектурные решения

### Физика
Портируется 1:1 из оригинала. Находится в `game/physics/`.
Не абстрагируется через движок — это игровая физика, не общая.

### Загрузка ресурсов
Портируется 1:1. Форматы файлов не меняются.
Ресурсы те же самые из оригинальной игры.

### Игровые объекты
Оригинал использует Thing (union + function pointers).
В новой версии — рассмотреть:
- Entity-Component System (ECS)
- Или просто C++ наследование (проще портировать)
- Решить после анализа как Thing используется в оригинале

### Система миссий (EWAY)
MuckyBasic в оригинале фактически не используется — вся миссионная логика реализована
через data-driven систему EWAY (EventPoints). Скриптовый движок не нужен.
Реализуется: C++ движок, читающий бинарный .ucm формат (условия/действия EventPoints).
