# Этап 2 — Удаление мёртвого кода ✅ ЗАВЕРШЁН

**Цель:** максимально очистить код от ненужного мусора без изменения чего-либо — ни логики,
ни имён, ни структуры. На выходе тот же код, только без балласта.

**Главное правило этапа: код переносится 1 в 1, без каких-либо изменений.**

## Что разрешено удалять

- `#ifdef PSX` / `TARGET_DC` / `VERSION_GLIDE` / `EDITOR` ветки и весь код внутри
- Папки редакторов: `Editor/`, `GEdit/`, `LeEdit/`, `SeEdit/`
- `Glide Engine/` — 3dfx Glide рендерер
- `MuckyBasic/`, `thrust/` — не используются в игре
- `PSXENG/` — PSX-специфичный рендеринг
- `Command.cpp` — legacy, заменён PCOM в финале
- `cam.cpp` — мёртвый (`#ifdef DOG_POO`), активна только `fc.cpp`
- `#if 0` блоки — мёртвый закомментированный код
- Любой другой код который **однозначно задокументирован** в KB как неиспользуемый,
  экспериментальный, не вошедший в релиз или выключенный

Запрещено:
- Переименовывать переменные, функции, типы, файлы
- Менять структуру файлов и папок
- Рефакторить что-либо (даже очевидное)
- Менять логику, порядок операций, алгоритмы

## Предупреждения

- **⚠️ Система инклюдов в проекте кривая — никогда не предполагай, всегда проверяй:**
  Инклюды нередко тянут лишнее неочевидным образом. Перед любым действием с файлом/папкой —
  grep по всему проекту, не полагаться на интуицию.

- **⚠️ Перед удалением любой папки или файла:** найти все `#include` без обёртки удаляемого
  `#ifdef`. Если такие есть — раскопать что используется. Если нужно для рантайма — вынести код.
  Только после этого удалять.

- **Минимальные правки для компиляции** — только если удаление сломало синтаксис окружающего
  кода. Допустимо: убрать осиротевший `else`, закрыть незакрытую скобку, убрать `#endif` без
  `#ifdef`. Недопустимо: переписывать логику, менять структуру, добавлять что-то новое.

## Порядок выполнения

1. Удаление ненужных папок и файлов с кодом ✅ **СДЕЛАНО**
2. Удаление посторонних файлов репозитория (инсталляторы, дистрибутивы, сборочный мусор) ✅ **СДЕЛАНО**
3. Удаление кода задокументированного в KB как неиспользуемый/не в финале:
   — По одному флагу/фиче за итерацию, компиляция после каждой ✅ **СДЕЛАНО**
   — Вырезанные фичи: `BIKE`, `FAST_EDDIE`, `USE_PASSWORD`, `SEWERS`, `TRUETYPE`, `OLD_CAM`/`CAM_OLD`, `DOG_POO`
   — Региональные/DRM: `EIDOS` и все региональные
   — Мёртвые компиляторы/платформы: `_MF_DOSX`, `__WATCOMC__`, `__DOS__`, `__WINDOWS_386__`
   — Glide, debug-флаги, developer joke flags — см. полный список в логе
4. Переход на Release-only кодовую базу — удаление всех `#ifdef _DEBUG`/`#ifdef DEBUG` блоков ✅ **СДЕЛАНО (итерация 33)**
5. Полное устранение условной компиляции — удаление всех `#ifdef`/`#ifndef`/`#if` (кроме include guards) ✅ **СДЕЛАНО**
6. Удаление файлов без `#include` и без упоминания в vcxproj
7. Удаление мёртвых сущностей (функции, переменные, типы, нигде не используемые)
8. Форматирование через clang-format (двойные пустые строки, отступы после массовых правок)
9. Финальная очистка пустых файлов

**Решение (2026-03-18):** Release-only кодовая база. Debug-код удалён навсегда.
При необходимости восстанавливать из `original_game/`.

**Флаги Release-конфигурации** (из vcxproj): `NDEBUG;_RELEASE;WIN32;_WINDOWS;VERSION_D3D;TEX_EMBED;FINAL`

## Правила работы на Этапе 2

**Итерационность — главное правило:**
- Каждую итерацию пользователь запрашивает отдельно. Клод **не запускает следующую итерацию сам**.
- После каждой итерации: компиляция → smoke-тест → коммит. Только тогда следующая итерация.
- **КРИТИЧНО:** после любого автоматического изменения файлов — СТОП. Дождаться компиляции от пользователя.
- Каждая итерация = один логический кластер удалений (одна папка, один флаг, один тип мёртвого кода)

**Читать KB перед работой:**
Перед началом каждой итерации читать соответствующий subsystem файл и/или раздел DENSE_SUMMARY.
