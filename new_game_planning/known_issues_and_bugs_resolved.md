| Hotplug DualSense↔Xbox не работает на лету (2026-04-17) | Комбинация **четырёх** накладывающихся багов, вскрытых при тестировании через F11 debug-панель. (1) Activity-скан DS в `gamepad_poll` был гейтован `s_is_dualsense` вместо фактического успеха `poll_dualsense()` — первый кадр хотплага терялся, touchpad click/mute/пальцы тачпада никогда не промотировали active (цикл шёл только до 16). (2) `gamepad_poll` имел fast-path "drain and ignore" для SDL ивентов пока DS primary — Xbox ADDED/REMOVED **терялись навсегда**. (3) На Bluetooth disconnect не детектился: `SDL_hid_read` молча возвращает 0 без ошибки, `ds_is_connected()` оставался `true` вечно (USB этого бага не имел — там read -1 на отдёргивании кабеля). (4) `SDL3_GamepadEvent` не содержал instance_id устройства — хендлер `REMOVED` тупо закрывал `s_gamepad` на **любом** ивенте (SDL отслеживает DS как гамепад даже когда hidapi-backend попросили не клеймить, REMOVED для DS убивал Xbox handle). Протестировано юзером во всех комбинациях USB/BT для обоих контроллеров. Детали → [`new_game_devlog/input_debug_panel.md`](../new_game_devlog/input_debug_panel.md) (Iterations 11-14). |

| DualSense adaptive trigger: щелчок не совпадает с выстрелом | **Полностью исправлено 2026-04-17** переходом на act-bit детект щелчка. История: изначально fire шёл раньше щелчка, так как использовался raw r2-threshold (FIRE_THRESHOLD=128). 2026-04-15 частичный фикс — FIRE_THRESHOLD=200 совпадает с точкой клика Weapon25, плюс rising-edge детектор. 2026-04-17 переписали `weapon_feel_evaluate_fire` на **прямой hardware-сигнал** — бит 4 байта 0x29 DualSense input report'а (act flag): fire на переходе `act 1→0` с `r2 > reset_threshold`. | DualSense adaptive trigger: rapid-tap выстрелы без щелчка / click-without-shot во время cooldown | **Полностью исправлено 2026-04-17** серией связанных изменений. Эволюция фикса: (1) act-bit детект щелчка — fire на прямом hardware сигнале 1→0. Убрало "shot without click" для slow taps. Но для rapid tap'ов hardware motor имеет natural rate limit ~300мс — клики пропадали в пользу threshold fallback'а без тактильного фидбека. (2) `fire_cooldown_ms` gate на mode=NONE после каждого выстрела — это не только убрало "click without shot" во время post-fire окна, но и **оказалось ключом к обходу ~300мс rate limit**: кратковременный сброс эффекта до NONE заставляет hardware motor перевзводиться быстрее natural self-rearm. Финальная настройка `pistol.fire_cooldown_ms=100` даёт тактильный клик на КАЖДОМ выстреле при любом темпе нажатий, rate близок к клавиатурному маш'ингу. Важное открытие про hardware — записано в [`dualsense_adaptive_trigger_facts.md`](../new_game_devlog/dualsense_libs_reference/dualsense_adaptive_trigger_facts.md) "Мотор можно форсить сбросить отправкой mode=NONE на ~100мс". Cooldown tracked локально в `weapon_feel` (wall-clock) — Person::Timer1 не использовался так как он нереально реализован (декрементируется только в STATE_MOVEING + reused для неожиданных целей). Детали и эволюция в [`weapon_haptic_and_adaptive_trigger.md`](../new_game_devlog/weapon_haptic_and_adaptive_trigger.md). |

Но это проявлялось только на **стенах с foundation** (`FHeight != 0`): `MakeFacetPointsCommon` ([`facet.cpp`](../new_game/src/engine/graphics/geometry/facet.cpp)) рисует нижний ряд таких стен не на `Y[0]`, а на **уровне земли из PAP-альтитуды** (`PAP_2HI(...).Alt << 3`) — на сложном рельефе (горы Target UC) foundation растягивается далеко вниз, и реальная видимая часть стены продолжает оставаться в кадре, тогда как AABB про foundation не знает (использует `Y[0]..Y[1]+Height*64`). Между визуальной геометрией и AABB возникает рассинхрон, и стена ошибочно отсекается. **Исправлено:** в обе AABB-конструкции (в `FACET_draw` и `FACET_draw_rare`) добавлено: при `FHeight != 0` опустить `y1` AABB до `min(PAP_2HI(v0).Alt << 3, PAP_2HI(v1).Alt << 3)`, если это ниже текущего `y1`. AABB только расширяется (никогда не сужается) — strictly conservative, не вносит регрессий для не-foundation стен. Перф-cost: 2 PAP-lookup + min, только для facet'ов с `FHeight != 0` (не каждая стена). Расследование шло через дамп clip-флагов всех p1-rejected facet'ов в файл по F6/F7 — увидели `clip_and = 0x04` (TOP) для двух пропадающих facet'ов и весь `world_y` диапазон выше камеры, пользователь подсказал «визуальная != физическая модель», и обнаружилось расхождение foundation render path vs AABB path. |
