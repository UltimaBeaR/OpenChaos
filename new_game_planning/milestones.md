## Фаза 3: Разработка

### Итерация 1 — Рабочая копия оригинала

**Цель:** скомпилировать и запустить игру из оригинального кода с минимальными изменениями.

**Стратегия:** копипаста 1:1, без изменения архитектуры и декомпозиции.
Старый компилятор (MSVC), старый DirectX, старая архитектура — всё как было.

**Что делается:**
- Копия кода из `original_game/`
- Выпиливание мёртвого и ненужного кода:
  - `#ifdef PSX` / `TARGET_DC` / `VERSION_GLIDE` / `EDITOR` ветки
  - Редакторы (`Editor/`, `GEdit/`, `LeEdit/`, `SeEdit/`)
  - Glide Engine
  - `MuckyBasic/`, `thrust/` — не используются в игре
  - `Command.cpp` — legacy, заменён PCOM в финале
  - `cam.cpp` — мёртвый (`#ifdef DOG_POO`), активна только `fc.cpp`
  - PSX-специфичный рендеринг (`PSXENG/`)
- Сохраняется: вся игровая логика, все PC-специфичные пути кода

**Что НЕ делается в итерации 1:**
- Не меняется компилятор
- Не меняется рендерер (DirectX остаётся)
- Не меняется архитектура
- Не добавляются модули

**Критерий успеха:** игра компилируется и запускается, можно пройти уровень.

### После итерации 1 — Исправление пре-релизных багов

Пре-релизная кодовая база содержит баги которых нет в финальной игре.
Фиксы делаются до итерации 2 чтобы иметь корректную базу для тестирования.

**Известные баги из анализа (требуют фикса):**

| Баг | Файл | Описание |
|-----|------|----------|
| Воскрешение гражданских | pcom.cpp | Телепорт на HomeX/HomeZ считается но не применяется — воскресают на месте смерти |
| Баг освещения | night.cpp | `dz×nx` вместо `dz×nz` в расчёте статических источников света |
| MAV INSIDE2 | mav.cpp | `INSIDE2_mav_inside()` → `ASSERT(0)`, NPC не ходят внутри зданий |
| MAV stair/exit | mav.cpp | `> 16` вместо `>> 16` — возвращает текущую позицию вместо целевой |
| Лестницы | interfac.cpp | `mount_ladder()` закомментирован |
| Crinkle | mesh.cpp | `return NULL` — отключён, но работает в финальной PC версии |
| Wibble воды | water.cpp | Закомментирован в пре-релизе, уточнить есть ли в финале |

**Дополнительные баги:** пользователь может принести фиксы из чужих веток —
проверить корректность по знанию оригинальной логики.

**Критерий:** поведение игры совпадает с финальной Steam PC версией.

### Итерация 2 — Новый рендерер + смена тулчейна

**Цель:** заменить DirectX рендерер на OpenGL/Vulkan через абстракцию, перейти на Clang.

**Стратегия:**
- Ввести интерфейс `IRenderer` на границе DDEngine → DDManager
- DDEngine (bucket sort, pages, polygon submission) остаётся
- DDManager заменяется реализацией `IRenderer` на OpenGL или Vulkan
- Параллельно: перенос мутаций стейта из рендерера в update фазу
  (конкретные места: `drawxtra.cpp`, `PYRO_draw_armageddon()`)
- Переход с MSVC на Clang — ожидаемо вылезут MSVC-специфичные вещи
  (`__int64`, `__declspec`, нестандартные расширения) — фиксить по ходу

**Выбор API:** OpenGL 4.1 vs Vulkan+MoltenVK — решить перед стартом итерации.
Подробно → `tech_stack.md`

**Критерий:** тестовая инфраструктура из итерации 1 проходит без регрессий.

### Итерация 3 — Кросс-платформа

**Цель:** убрать Windows-специфичный код.

- `WinMain` → SDL3 main
- Windows API вызовы → SDL3 / std::
- `DIManager.cpp` (DirectInput) → SDL3 input
- Звук: MSS32 → OpenAL Soft (или miniaudio)

**Критерий:** компилируется и работает на Linux и macOS.

---

### Итерация 4 — Модернизация кода

**Цель:** современная кодовая база.

- `#include` хедеры → C++23 modules
- Глобальные массивы → инкапсуляция
- C-style code → идиоматичный C++23

**Примечание:** модули делаются после смены компилятора на Clang —
поддержка модулей в MSVC хуже. Ожидаемы проблемы с макросами (не экспортируются из модулей).

**Критерий:** кодовая база компилируется с модулями, тесты проходят.
