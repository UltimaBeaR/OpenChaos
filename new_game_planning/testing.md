# Тестирование

Тестовая инфраструктура создаётся после Итерации 1 (рабочая копия + баг-фиксы),
до Итерации 2 (замена рендерера). Цель — иметь страховочную сетку для всех
последующих изменений.

---

## 1. Автоматические unit-тесты

Тестируют чистую логику без рендера, ввода и файловой системы.
Можно запускать headless, быстро, в CI.

### Физика и математика

- Формула урона от падения: `dmg = (-DY - 20000) / 100`, граничные случаи
- Формула урона от взрыва: `hitpoints = maxdmg × (radius - dist) / radius`
- Урон от дробовика: `300 - dist`, saturation
- Гравитация транспорта: `VelY += GRAVITY` per tick
- Fixed-point арифметика: 32.8, 15.15, 16.16 форматы
- Матричные операции (CMatrix33 декомпрессия, `UCA_Lookup`)
- `QuatSlerp()` — граничные случаи (dot > 0.95, антиподы)
- `Arctan()` через `AtanTable[256]`
- Friction формула: `((vel << friction) - vel) >> friction`

### Collision и физика персонажа

- `slide_along()` — базовые случаи (стена по X, стена по Z, угол)
- `find_face_for_this_pos()` — порог кандидата 0xa0, fallback 0x50
- `plant_feet()` — возвращаемые значения (-1, 0, 1)
- Граница карты — выход за пределы возвращает 0

### Боевая система

- Таблица комбо: Punch COMBO1/2/3 (10/30/60), Kick, Cross-комбо, Knife, Bat
- Формула блока AI: `base = 60 + GET_SKILL×12`, cap
- Grapple дистанции: PISTOL_WHIP(75), STRANGLE(50), HEADBUTT(65), GRAB_ARM(60)
- Hit chance: `base = 230 - abs(Roll)>>1 - Velocity + модификаторы`, min=20/256

### EWAY система

- Вычисление условий (EWAY_COND_*) на синтетических данных
- Режимы активации: ALWAYS, WHILE, WHILE_TIME, TIME
- COUNTDOWN decrement логика
- Spawn координаты: mapsquare × 256 → world, Yaw << 3 / >> 3

### MAV навигация

- `MAV_can_i_walk()` — Bresenham на маленьких сетках
- `MAV_precalculate()` на синтетической 4×4 сетке
- Пороги высот: dh=0–2 → GOTO; 2–4 → PULLUP; 1–2 → JUMP и т.д.
- `MAV_turn_movement_on/off()` — потеря caps кроме GOTO (известный баг, поведение 1:1)

### Форматы файлов

- Парсинг заголовка `.ucm` (1316 байт, версии 1–9+)
- Парсинг `.wag` v3 (mission_name, complete_point, stats, hierarchy, best_found)
- PAP_Hi cell: Texture(UWORD), Flags(UWORD), Alt(SBYTE×8)
- `best_found` анти-фарм: только дельта сверх предыдущего рекорда засчитывается

### Прочее

- `ROAD_is_road()`, `ROAD_is_middle()` — маркеры дорог
- Tripwire детекция: point-to-line distance < 0x20
- HyperMatter stationary threshold = 0.25
- Grenade таймер: 1920 тиков = 6 сек при 20fps

## 2. Детерминированный replay

**Идея:** записать последовательность входных данных (Input bitmask per frame) +
фиксированный RNG seed → воспроизвести → сравнить финальный стейт.

**Что нужно для детерминизма:**
- Фиксированный RNG seed (заменить `rand()` на seeded версию)
- `TICK_RATIO = 256` (константа, без зависимости от реального времени)
- Отключить ambient sound таймеры (они используют `rand()` для интервалов)

**Что сравниваем:**
- Позиции всех Things (WorldPos X/Y/Z)
- Health всех персонажей
- Состояния (State, SubState) игрока
- Количество патронов, предметов

**Применение:** после каждой итерации прогнать записанные сценарии →
если стейт совпадает, значит логика не сломана.

**Готовые сценарии для записи:**
- Пробежка по тестовому уровню с активацией всех механик
- Перестрелка с несколькими NPC
- Поездка на машине с авариями
- Прохождение первой миссии

## 3. Тестовый уровень

**Идея:** специальный уровень в котором сосредоточены все игровые механики.
Для ручного тестирования и как среда для replay сценариев.

### Как создать

Редактор карт есть в оригинальной кодовой базе (`Editor/`, `GEdit/`) — замороченный,
но рабочий. Альтернативно — создать уровень программно через прямое заполнение
`.ucm` + `.iam` структур (форматы задокументированы).

**Нужно исследовать:** как именно запустить редактор из оригинальной кодовой базы.
Возможно есть готовые тестовые уровни в `original_game/fallen/Debug/levels/`.

### Что должно быть на уровне

**Движение и акробатика:**
- Открытая площадка для бега, прыжков, скольжения
- Стена для лазания (CLIMB_OVER, CLIMB_UP, CLIMB_DOWN)
- Карниз для свисания (DANGLING, PULL_UP, DROP_DOWN)
- Лестница (MOUNT, ON_LADDER, CLIMB_OFF_TOP/BOT)
- Трос/зипвайр (DANGLING_CABLE, FORWARD/BACKWARD)
- Уступ для FIRE_ESCAPE механики
- Перепад высот для fall damage проверки (DY ≤ -30000 → 250 dmg)

**Бой:**
- Несколько манекенов/NPC для проверки всех комбо
- Punch COMBO1/2/3, Kick COMBO1/2/3, Cross-комбо
- Knife (30/60/80), Bat (60/90)
- Grapple зоны: PISTOL_WHIP, STRANGLE, HEADBUTT, GRAB_ARM, SNAP_NECK

**Оружие:**
- Пикапы всех типов: Pistol(15), Shotgun(8), AK47(30), Grenade(3), Baseball Bat, Knife
- Взрывчатка + таймер (1600 тиков)
- Мишени для проверки урона на разных дистанциях (Shotgun: 300-dist)

**NPC и AI:**
- CIV (гражданский, проверка WANDER и воскрешения)
- COP (арест, погоня)
- THUG (агрессия, патруль)
- MIB (каждый кадр проверяет видимость Darci, invulnerable)
- Проверка FOV: близко (<192 units) = 123°, далеко = 74°

**Транспорт:**
- CAR, POLICE, AMBULANCE, JEEP (по одному каждого)
- Трасса для проверки скорости (VEH_SPEED_LIMIT=750)
- Препятствия для проверки повреждений (6 зон × 4 уровня)
- Скользкая поверхность для skid (SKID_START=3 кадра, SKID_FORCE=8500)

**Интерактивные объекты:**
- Двери (обычные 180° и 90° варианты)
- Бочки (все 4 типа: NORMAL, CONE, BURNING, BIN)
- Платформа с маршрутом (PLAT_STATE_GOTO/PAUSE)
- Лужи для проверки отражений
- Провод/кабель под током

**Эффекты:**
- Зона пожара (IMMOLATE: -10 HP/frame)
- Граната (таймер 1920 тиков = 6 сек)
- Бочка для взрыва

**Освещение:**
- День и ночь (переключение)
- Динамический источник света (DLIGHT: NORMAL/BROKEN/PULSE/FADE)
- Статические источники

**Интерьеры:**
- Здание с дверями (открытие/закрытие, MAV sync)
- Интерьер со лестницей
- Склад (WARE)

**Миссионная логика (EWAY):**
- EventPoint с PROXIMITY условием
- Countdown таймер
- Спавн NPC по триггеру
- Диалог (CONVERSATION action)

## 4. Ручной чеклист

Выполнять после каждой итерации с изменениями. Сохранить как отдельный файл
`testing_checklist.md` когда придёт время.

### Движение
- [ ] Ходьба, бег, приседание, скольжение
- [ ] Прыжок с места, с разбега, grab прыжок
- [ ] Свисание с карниза, подтягивание, спрыгивание
- [ ] Лазание по стене (вверх, вниз, влево, вправо, через)
- [ ] Лестница (вверх, вниз, выход сверху и снизу)
- [ ] Трос/зипвайр (вперёд, назад)
- [ ] Падение с разных высот (без урона, с уроном, мгновенная смерть)
- [ ] Plunge анимация (DY < -12000)

### Бой
- [ ] Комбо кулаками (3 удара → KO)
- [ ] Комбо ногами (3 удара → KO)
- [ ] Cross-комбо (кулак+нога)
- [ ] Блок (AI блокирует удары)
- [ ] Grapple все 4 варианта
- [ ] SNAP_NECK (смерть от захвата)
- [ ] Fall damage на NPC

### Оружие
- [ ] Пистолет: стрельба, перезарядка, автоприцеливание
- [ ] Дробовик: урон зависит от дистанции
- [ ] АК-47: очередь
- [ ] Граната: бросок, отскок, взрыв через 6 сек
- [ ] Взрывчатка: установка, таймер 1600 тиков
- [ ] Нож, бита — меле комбо

### NPC и AI
- [ ] Гражданские ходят по маршруту (WANDER)
- [ ] Cop реагирует на преступление
- [ ] Thug атакует
- [ ] MIB засекает Darci, вызывает подмогу
- [ ] NPC слышат выстрелы (радиус 2560 units)
- [ ] NPC реагируют на шум (банка колы: UNUSUAL)
- [ ] Арест: RedMarks накапливаются, 3 → game over

### Транспорт
- [ ] Посадка/выход из всех типов машин
- [ ] Разгон до максимума (750 units)
- [ ] Задний ход (300 units)
- [ ] Занос (SKID)
- [ ] Повреждения (6 зон)
- [ ] Дым при >50% компрессии подвески
- [ ] Наезд на NPC

### Интерактивные объекты
- [ ] Дверь открывается/закрывается (180° и 90°)
- [ ] MAV обновляется при открытии двери
- [ ] Бочка катится, горит, взрывается
- [ ] Платформа движется по маршруту
- [ ] Стоять на движущейся платформе (WMOVE)
- [ ] Стоять на крыше машины (WMOVE)

### Визуал и эффекты
- [ ] Частицы взрыва (POW, PYRO)
- [ ] Огонь (IMMOLATE: видимый урон -10 HP/frame)
- [ ] Искры (SPARK)
- [ ] Следы шин (TYRE_SKID)
- [ ] Кровь (при VIOLENCE=TRUE)

### Освещение

### Сохранение/загрузка
- [ ] Сохранение в слот 1, 2, 3
- [ ] Загрузка сохранения

### Фронтенд

## Приоритет создания тестов

1. **Сначала:** unit-тесты физики и формул урона — быстро, высокая ценность
2. **Потом:** парсинг форматов файлов — нужен для загрузки ресурсов
3. **Потом:** детерминированный replay — записать до итерации 2
4. **Параллельно:** исследовать как создать тестовый уровень через редактор или программно
