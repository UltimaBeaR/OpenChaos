# Технический стек

## Статус: Предварительный — пересмотреть после анализа оригинала

## Язык

**C++23** с модулями (C++20 modules)

**Причины:**
- Физика и логика портируются механически (нет концептуального перевода)
- Лучшая экосистема для нативного геймдева
- C++23 modules убирают боль с #include хедерами
- Не нужно учить новый язык в процессе сложного проекта

**Не брать:**
- MSVC/Visual C++ — привязка к Windows
- Rust — другая парадигма, портировать C-логику болезненно

## Компилятор и сборка

- **Компилятор:** Clang (лучшая поддержка C++23 modules, кросс-платформенный)
- **Сборка:** CMake
- **Пакеты:** vcpkg

## Платформы

Windows, Linux, macOS — с первого дня.

## Окно и ввод

**SDL3**

**Причины:**
- Нативная поддержка геймпада (важно для будущего)
- Кросс-платформенно
- Хорошо зарекомендован

## Графика

**Предварительно: OpenGL 4.6**

Пересмотреть после анализа рендерера оригинала.

**Аргументы за OpenGL:**
- Проще стартовать и портировать
- Достаточно для воспроизведения графики 1999 года
- Хорошая кросс-платформенная поддержка

**Аргументы за Vulkan:**
- Современнее, лучше для будущего
- Более низкоуровневый контроль
- Сложнее стартовать

**Решение:** начать с OpenGL для быстрого старта, возможна миграция на Vulkan позже.

## Float precision

**Проблема:** компиляторы могут использовать 80-bit extended FPU precision,
что даст другие результаты чем оригинал (32-bit float).

**Решение:** флаги компилятора `-mfpmath=sse -msse2` (принудительно SSE = строго 32-bit).
Добавить в CMakeLists.txt.

**Важно для:** физики — она портируется 1:1 и должна давать те же результаты.
