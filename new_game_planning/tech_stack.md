# Технический стек

## Статус: Пересмотрен после анализа оригинала (2026-03-14)

## Язык

**C++23** с модулями (C++20 modules)

**Причины:**
- Физика и логика портируются механически (нет концептуального перевода)
- Лучшая экосистема для нативного геймдева
- C++23 modules убирают боль с #include хедерами
- Не нужно учить новый язык в процессе сложного проекта

**Не брать:**
- MSVC/Visual C++ — привязка к Windows
- Rust — другая парадигма, портировать C-логику болезненно

## Компилятор и сборка

- **Компилятор:** Clang (лучшая поддержка C++23 modules, кросс-платформенный)
- **Сборка:** CMake
- **Пакеты:** vcpkg

## Платформы

Windows, Linux, macOS — с первого дня.

## Окно и ввод

**SDL3**

**Причины:**
- Нативная поддержка геймпада (важно для будущего)
- Кросс-платформенно
- Хорошо зарекомендован

## Графика

**На рассмотрении: OpenGL 4.1 vs Vulkan**

Платформы: Windows, Linux, macOS — все три с первого дня.

**Ситуация с API по платформам:**

| API | Windows | Linux | macOS |
|-----|---------|-------|-------|
| OpenGL 4.6 | ✓ | ✓ | ✗ (deprecated, max 4.1) |
| OpenGL 4.1 | ✓ | ✓ | ✓ (deprecated, но работает) |
| Vulkan | ✓ | ✓ | через MoltenVK |

### OpenGL 4.1

**Аргументы за:**
- Проще стартовать, меньше boilerplate.
- 4.1 достаточно для воспроизведения оригинала (Painter's Algorithm, vertex lighting, compute не нужен).
- Работает на macOS сейчас.

**Аргументы против:**
- Deprecated на macOS с 2018 года — Apple может удалить в любой момент.
- Ограниченные возможности для визуальных улучшений (нет compute shaders в 4.1).
- OpenGL в целом в maintenance mode с 2017 — официальный преемник Khronos это Vulkan.

### Vulkan + MoltenVK

**Аргументы за:**
- Официальный преемник OpenGL от Khronos Group.
- Единый API на всех трёх платформах (MoltenVK транслирует в Metal на macOS).
- Future-proof: новые фичи, ray tracing, modern rendering.
- MoltenVK активно поддерживается и используется в продакшене (Steam, Dota 2 и др.).
- Если захочется улучшить графику позже — не нужно переписывать рендерер.

**Аргументы против:**
- Значительно больше boilerplate для старта.
- Сложнее в разработке и отладке.

**Вывод:** оба варианта жизнеспособны. Vulkan future-proof, OpenGL 4.1 проще стартовать.
Финальный выбор — при старте Фазы 3.

## Звук

**На рассмотрении: OpenAL Soft vs miniaudio**

Оригинал использует Miles Sound System с 3D позиционным звуком (`MFX_play_xyz`).
Нужна кросс-платформенная библиотека с 3D spatializer.

### OpenAL Soft (предпочтительный кандидат)

**Аргументы за:**
- Нативный 3D позиционный звук — прямой аналог MSS32.
- Зрелый, хорошо проверенный в игровой разработке.
- Кросс-платформенно (Windows через OpenAL Soft, Linux, macOS).
- Активно поддерживается (OpenAL Soft от kcat).

**Аргументы против:**
- Чуть больше настройки чем miniaudio.
- Apple deprecated свой OpenAL, но OpenAL Soft решает это.

### miniaudio

**Аргументы за:**
- Header-only, тривиальная интеграция.
- Кросс-платформенно.

**Аргументы против:**
- 3D spatializer базовый — менее зрелый чем OpenAL.

**Вывод:** OpenAL Soft вероятно лучше для 3D звука, но оба жизнеспособны.
Финальный выбор — при старте Фазы 3.


## Float precision

**Проблема:** компиляторы могут использовать 80-bit extended FPU precision,
что даст другие результаты чем оригинал (32-bit float).

**Решение:** флаги компилятора `-mfpmath=sse -msse2` (принудительно SSE = строго 32-bit).
Добавить в CMakeLists.txt.

**Важно для:** физики — она портируется 1:1 и должна давать те же результаты.

## Анимационная система

**Никаких библиотек не нужно.**

Оригинал использует vertex morphing (не скелетную анимацию): линейная интерполяция
между двумя наборами вершин (tween ∈ [0,256]). Реализуется вручную в шейдере или CPU.

## Архитектурное требование: рендерер ≠ game state

В оригинале рендерер мутирует game state (`drawxtra.cpp` модифицирует ammo, создаёт
PYRO/SPARK прямо внутри draw pass). В новой версии — строгое разделение:
рендерер только **читает** state, никогда не пишет.

## Что НЕ использовать

- DirectX — привязка к Windows
- Visual C++ / MSVC — привязка к Windows
- Unity/Godot — потеря низкоуровневого контроля над форматами ресурсов
- Lua — пользователь не хочет
